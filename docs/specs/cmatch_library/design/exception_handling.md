# 异常处理

## 实体数据修复

单个实体的数据是一致的，但是多个实体的数据可能不一致，在 `BuildSeason` 阶段，需要对数据修复：

- 对每种赛季类型，将实体数据分为两类，一类是当前时间在赛季时间范围内的，另一类是当前时间不在赛季时间范围内的。
- 对不在当前赛季的：
  - 按照分组进行结算，计算排名时，需要将已经结算同一分组的凭据也算在内，但是不修改其数据。
  - 计算新的段位
  - 加入到当前赛季的分组，如果分组大小不够，创建新的分组。

## 赛季时间修复

配置的赛季时间可能和凭据上保存的赛季时间不一致，但是判断是在同一赛季时，使用配置的赛季时间为准，并且修复凭据上的赛季时间。
