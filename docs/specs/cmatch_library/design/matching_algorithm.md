# 匹配算法

## 同一赛季判断

- 当前时间 `now_time`。
- 赛季时间由 `begin_time` 和 `end_time` 定义，闭开区间。
- `now_time` >= `begin_time` 且 `now_time` < `end_time` 时，表示当前时间在赛季中。
- 传递给`TicketManager`的`now_time`，必定在`season_time`参数的范围内。
- 当 `now_time` 不在凭据上的赛季时间范围内时，表示凭据不在赛季中。

## 赛季结算

- 将同一分组的凭据根据积分从高到低进行排序。积分相同时，按凭据 ID 升序作为第二排序键，确保排名唯一。
- 按排序后的顺序依次计算排名，排名从 1 开始。
- 排名百分比为该凭据排名除以分组内凭据总数。
- 保存赛季类型和时间、段位、分组编号、排名、排名百分比、积分等结算信息到凭据的结算数据中。

## 分组 ID 分配

- 分组 ID 由两部分组成：高 32 位为分组中第一个凭据的 `zone_id`，低 32 位为一个自增的编号，组合成一个 64 位整数。
- 当实体加载完成后，获取所有凭据上的分组 ID，包括结算数据里的分组 ID，计算最大的自增编号。
- 当为一组凭据分配分组 ID 时，使用第一个凭据的 `zone_id` 作为高位，使用递增后的自增编号作为低位，生成分组 ID。
- 此算法可以正确处理合服的情况，保证分组 ID 的唯一性。

## 段位升降

切换赛季时，需要根据配置 `grade_info` 来计算每个赛事类型的下一个赛季段位。

- 对当前段位的 `GradeInfo`，先检查升段条件。若配置了 `promote_rank_percent`，则当结算排名百分比小于等于该值时升段；否则若配置了 `promote_rank`，则当结算排名小于等于该值时升段；两者均配置为 0 表示不升段。
- 若未升段，再检查降段条件。若配置了 `demote_rank_percent`，则当结算排名百分比大于该值时降段；否则若配置了 `demote_rank`，则当结算排名大于该值时降段；两者均配置为 0 表示不降段。
- 升段时目标段位为 `GradeInfo.next_grade`，降段时目标段位为 `GradeInfo.prev_grade`。

## 赛季分组

- 将同一段位的凭据使用 `std::shuffle` 配合外部传入的随机数引擎打乱顺序，分组大小由配置 `group_size` 决定。`group_size` 为 0 时表示该段位只有一个分组。
- 按打乱后的顺序依次取 `group_size` 个凭据为一个分组，不足 `group_size` 的剩余凭据也作为一个分组。
- 为每个分组分配一个唯一的分组 ID。
- `Initialize` 中，仅对不在当前赛季内的凭据按段位重新分组；在当前赛季内的凭据保持原分组不变。

## 统一填充与新建分组

`Initialize` 对不在当前赛季内的凭据结算并计算新段位后，会将其与新加入的凭据统一收集。
同段位的待分组凭据优先填入当前赛季内未满的旧分组（通过未满组队索引快速定位），填满后从未满索引中移除；剩余凭据再统一创建新分组，从而减少分组碎片并提升效率。

## 分组索引

`TicketManager` 维护以下索引以加速查询与后续扩展：

- `<season_type, grade> -> group_ids`：按赛季类型与段位快速定位所有分组。
- `group_id -> ticket_ids`：按分组 ID 快速获取成员凭据。
- `<season_type, grade> -> group_ids`（未满）：快速定位未满的分组，用于后续新凭据加入时优先填充。

`GetGroupMembers` 接口通过 `group_id -> ticket_ids` 索引直接返回成员，无需遍历全部凭据。
